O Android Studio é um IDE baseado no IntelliJ cujo foco é o desenvolvimento de aplicações para Android. Neste codelab veremos como fazer a sua instalação no ambiente Windows.

Nota. O Android Studio também pode ser instalado nos ambientes MacOS e Linux, embora não tenhamos planos de abordar estas instalações neste codelab.

O primeiro passo é visitar a página oficial do IDE para obtê-lo.

Clique a seguir para visitar a página oficial. A seguir, clique para fazer o download. Observe que as opções, leiaute da tela etc poderão ser diferentes quando você acessar a página.

Download Android Studio

A seguir, você terá de aceitar os termos de uso. Role o texto - de preferência lendo com atenção -, marque a caixa de concordância e clique para fazer o download.

Uma vez que tenha feito o download do IDE, clique duas vezes sobre o instalador para iniciar a instalação. Ele provavelmente terá sido armazenado na sua pasta de Downloads. Pode ser a padrão do sistema operacional ou aquela que você tiver configurado em seu navegador. Vai depender da sua máquina. Na figura a seguir, o arquivo se encontra na pasta de Downloads padrão do sistema operacional.

Depois de clicar duas vezes, o mecanismo de permissões do sistema operacional entrará em cena e pedirá a sua autorização para executar o arquivo. Clique em sim.

A seguir, clique em Next.

Mantenha as opções marcadas e clique em Next mais uma vez.

Na tela seguinte, você deve escolher o diretório de instalação do Android Studio. É recomendável manter a opção padrão. Clique em Next depois de fazer a sua escolha.

Depois disso, o assistente faz uma pergunta sobre a criação de um atalho no menu iniciar. Mantenha a opção padrão e clique em Install.

Quando a instalação terminar, clique em Next.

A seguir, mantenha a caixa Start Android Studio marcada e clique em Finish. O Android Studio será aberto automaticamente.

A próxima tela te dará a oportunidade de importar configurações de eventuais instalações passadas do Android Studio. Se esta for a primeira vez que está fazendo a instalação você não as terá. Basta manter a opção Do not import settings marcada e clicar em OK.

A seguir, você precisa decidir se deseja permitir que o Android Studio colete informações sobre seu funcionamento e as envie periodicamente ao Google para análise. Segundo a mensagem, as informações são anônimas. Faça a sua escolha. Neste exemplo, estamos optando por enviar as informações.

Agora você começará a fazer algumas configurações sobre o funcionamento do IDE. Apenas clique em Next.

A seguir você pode escolher configurações padrão ou personalizadas. Vejamos o que podemos personalizar. Escolha Custom e clique em Next.

O Android Studio faz uso de uma ferramenta chamada Gradle para gerenciar as dependências do projeto, controlar o processo de build e outras diversas coisas. O que precisamos saber dele agora é que ele - o Gradle - depende de uma implementação do Java para funcionar. Precisamos escolher qual implementação usar. Observe que o Android Studio já inclui uma. Estamos apenas mantendo a padrão. Clique em Next.

A próxima configuração permite escolher entre tema escuro (Darcula) ou claro (Light). Faça a sua escolha e clique em Next. Neste exemplo, estamos escolhendo Darcula.

A seguir, escolhemos ferramentas importantes para o funcionamento do ambiente. Uma delas viabiliza a aceleração do emulador utilizando hardware. A disponibilidade desta opção depende das configurações de seu computador. Uma outra, Android Virtual Device, é um dispositivo virtual que poderemos colocar em execução para testar os aplicativos. Faça as escolhas destacadas e clique em Next.

Dependendo de suas configurações, pode ser que o ambiente o instrua a escolher a quantidade de memória RAM a ser utilizada para fazer a aceleração da execução do emulador. Faça a sua escolha (escolher a quantidade recomendada pode ser uma boa ideia) e clique em Next.

A seguir, será preciso clicar em Next mais uma vez.

A tela a seguir permite verificar os itens que serão instalados e requer mais um clique em Next. Observe.

Agora precisamos aceitar as licenças dos itens que estamos instalando. Novamente, os itens que aparecem aqui dependem das configurações de sua máquina. É possível que você tenha opções diferentes, mas o passo a passo para aceite das licenças é essencialmente o mesmo. Começamos aceitando a licença do android-sdk.

Depois, aceitamos a licença do android-sdk-preview.

Também aceitamos a licença do intel-android-extra. Aproveite e já clique em Finish.

Agora a coisa pode enrolar um pouco, pois o download é grande. Aguarde. Quando tudo terminar, clique em Finish.

Agora vamos fazer a instalação dos seguintes itens.

  • Android SDK Command-line Tools O SDK do Android possui diversas ferramentas que podem ser executadas na linha de comando e que viabilizam o desenvolvimento e "build" de aplicações Android. O próprio Android Studio faz uso delas.
  • Google USB Driver Usuários Windows que desejam testar seus aplicativos em dispositivos reais conectados via USB, precisam fazer a instalação deste driver.
  • Google Play Services Dá acesso a diversas APIs, como como o uso de mapas, autenticação, Google Wallet (para transações on-line) entre outros.

Nota. Se desejar saber mais sobre as ferramentas de linha de comando, visite https://developer.android.com/tools#tools-sdk

Para fazer a instalação, comece clicando em More Actions >> SDK Manager.

Clique em SDK Tools e marque os itens mencionados. Clique em OK.

Clique em OK novamente.

Quando o download e instalação terminarem, clique em Finish.

Outros ambientes de desenvolvimento - Flutter, por exemplo - podem fazer uso das ferramentas que instalamos, sem necessariamente utilizar o Android Studio. Para tal, precisamos configurar alguns diretórios como variáveis de ambiente. Primeiro, vamos indicar onde se encontra a instalação do Android SDK. Clique uma vez mais em More Actions >> SDK Manager.

Copie o endereço do Android SDK.

Agora vamos criar uma variável de ambiente chamada ANDROID_HOME. Para tal, clique no botão iniciar do Windows e busque por env. Clique em Editar as variáveis de ambiente para a sua conta.

Clique em Novo para criar uma variável de ambiente para o seu usuário.

O nome dela deve ser ANDROID_SDK. O valor deve ser o endereço que você copiou do SDK Manager. Clique em OK.

A seguir, encontre a variável de ambiente Path de seu usuário. Clique uma vez sobre ela e clique em Editar.

Adicione as seguintes variáveis. Ao final, clique em OK e OK novamente.

Cuidado. Os endereços utilizados incluem versões específicas. Antes de fazer as suas configurações, visite os seus diretórios de instalação e certifique-se de utilizar as versões corretas.

Cuidado. Lembre-se de fechar todos os terminais que eventualmente já estivessem abertos e abra um novo.

Para ter certeza de que tudo deu certo, use

sdkmanager --version

Veja o resultado esperado. Claro, a sua versão pode ser diferente.

Teste também o comando

avdmanager list avd

avd significa "Android Virtual Device"> Ou seja, o avdmanager é um gerenciador de dispositivos virtuais Android. Dispositivos que podemos colocar em execução usando o emulador. O comando que utilizamos deve exibir os AVDs que já possuímos. Veja uma possível saída. A sua pode ser diferente.

Observe que o avd exibido possui um nome. Neste caso, seu nome é Pixel_3a_API_34_extension_level_7_x86_64. Copie esse nome para usarmos a seguir. Podemos colocar um emulador em execução, apenas para testar, com

emulator -avd Pixel_3a_API_34_extension_level_7_x86_64

Veja o resultado esperado.

Como esse foi apenas um teste, você pode fechar o emulador. Ele consome muitos recursos da máquina.

OK! Aparentemente todas as configurações foram feitas corretamente!

Façamos um pequeno teste para nos certificarmos de que a instalação foi feita corretamente. Neste teste, vamos criar uma aplicação Hello, World para Android e a executaremos num emulador. Comece clicando em New project.

A seguir, escolha Empty Activity e clique em Next.

Preencha os campos como a seguir e clique em Finish.

Na tela seguinte, o Android Studio mostrará o download e configuração de mais algumas ferramentas. Essa parte também pode demorar um pouco. Quando terminar, clique em Finish.

É possível que o Android Studio mostre uma janela com novidades (Whats new) na direita. Você pode averiguar o conteúdo se desejar e, depois, minimizar a janela. Observe.

Agora você deve estar diante do código da aplicação que acaba de ser criada. Trata-se de uma aplicação para Android com uma única tela que exibe o texto Hello, World. Observe que, para executar a aplicação, você deve esperar o Gradle terminar o seu processamento. Quando ele terminar, o botão para execução da aplicação deverá estar habilitado. Veja.

Como é a primeira vez que o emulador está sendo colocado em execução, esse processo também pode demorar alguns minutos. Veja o resultado esperado.

Parabéns! Você acaba de instalar o Android Studio e está pronto para começar o desenvolvimento de aplicações utilizando linguagens de programação como Java e Kotlin. Esta instalação também viabiliza o desenvolvimento de aplicações utilizando diferentes frameworks, como o Flutter e outros.
